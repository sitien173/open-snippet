# openmacro — Design

- **Date:** 2026-05-29
- **Status:** Confirmed design, ready for `writing-plans`.
- **Target:** Windows 10/11 desktop, single-user.
- **Repurposed template features:** Quick Pane (global-shortcut floating window) → our dynamic-form popup. Command Palette (Cmd+K) → snippet picker / quick search. Preferences system → our `config.toml` equivalent (shell consent, pause state, sync, autostart, length caps).

## Goals

A global text-expansion app for Windows. User defines abbreviations; when typed in any focused app, they are silently replaced with the expanded text. Supports:

1. Plain multi-line text and code snippets.
2. A cursor-placement token (`$|$`) so the caret lands at a marked position after insertion.
3. Built-in variables: date/time, clipboard, cursor, shell-command output.
4. Dynamic forms — a snippet can prompt the user for field values via a popup, then expand using those values.

Plus: tray icon, autostart with Windows, and cloud sync.

## Non-goals (MVP)

- Per-app allow/deny rules.
- Regex triggers (literal triggers only in MVP).
- Cross-platform (Windows only for MVP; architecture leaves the door open).
- First-party hosted sync backend (git-backed sync only — see Section 5).

## 1. Architecture overview

Single-process Tauri app, layout inherited from the template. One dedicated OS thread owns the global keyboard hook (`WH_KEYBOARD_LL`); a Tokio runtime owns everything else. Webviews are lazy — settings hidden by default, form popup opened on demand via the template's existing multi-window scaffold.

```
┌─────────────────────────────────────────────────────────┐
│  openmacro.exe  (Tauri host, single process)            │

│  │ src-tauri/       │◄──►│ src/ (React UI)          │  │
│  │ (Rust)           │    │  • Settings window       │  │
```

**Repo layout (after template fork):**

```
openmacro/
  src-tauri/                # Rust — single crate from template
    src/
      lib.rs                # template entry
      engine/               # NEW — orchestrator, owns hook→match→inject
      store/                # NEW — YAML loader, file watcher
      matcher/              # NEW — Aho-Corasick, buffer, word boundary
      hook/                 # NEW — WH_KEYBOARD_LL thread, ring buffer
      inject/               # NEW — clipboard paste + SendInput fallback
      expand/               # NEW — placeholder resolvers
      form/                 # NEW — form state machine, focus capture
      sync/                 # NEW — git agent, credential storage
      commands/             # tauri-specta commands (template pattern)
  src/                      # React 19 + shadcn/ui
    routes/
      settings/             # snippet list, editor, sync config, prefs
      form/                 # form popup (mounted in Quick Pane window)
      picker/               # snippet picker (Cmd+K) — bonus
    lib/api.ts              # auto-generated by tauri-specta
  src-tauri/tauri.conf.json
  package.json, vite.config.ts, etc. (from template)
```

**Key boundaries:**
- Hook thread: no I/O, no allocation in hotpath. Ring-buffers last N chars, signals matcher.
- YAML folder is authoritative; in-memory indexes are caches rebuilt on file-watcher events (debounced 200 ms).
- Forms render in the **repurposed Quick Pane window** — already a borderless, always-on-top, global-shortcut-driven floating webview in the template. We just change its content route and trigger source (snippet trigger, not Cmd+Shift+.).
- Sync agent works on the YAML folder as a working tree; never touches the matcher.
- App prefs (pause state, autostart, shell consent, sync remote URL, max length) live in the template's preferences system (Rust-side persistence + React hooks) — no separate `config.toml` needed.

## 2. Data model & YAML schema

**Folder layout under `%APPDATA%\openmacro\snippets\`:**

```
snippets/
  default.yaml          ← shipped starter pack
  personal.yaml
  work/
    email.yaml
    sql.yaml
  shared/               ← synced folder (see Section 5)
    team.yaml
```

Any `.yaml` under the root is loaded recursively. Folder = logical grouping only; no inheritance.

**Schema example covering all four MVP features:**

```yaml
version: 1
matches:
  - trigger: ";sig"
    replace: |
      — Alex Chen
      alex@example.com

  - trigger: ";now"
    replace: "It is {{date:%Y-%m-%d %H:%M}} right now."
    vars:
      - name: date
        type: datetime           # datetime | clipboard | cursor | shell | form

  - trigger: ";for"
    replace: |
      for (int {{var}} = 0; {{var}} < {{limit}}; {{var}}++) {
          $|$
      }
    vars:
      - { name: var,   type: form, label: "Loop var", default: "i" }
      - { name: limit, type: form, label: "Limit" }

  - trigger: ";head"
    replace: "Current HEAD is {{sha}}"
    vars:
      - name: sha
        type: shell
        cmd: ["git", "rev-parse", "--short", "HEAD"]
        timeout_ms: 1500
```

**Decisions:**
- Literal-string triggers, Aho-Corasick matched. Regex deferred.
- `{{name}}` placeholders reference `vars`. Unknown name → loud failure (tray notification).
- `$|$` is the single cursor token; stripped before injection, caret repositioned after.
- `vars` is ordered. Form vars become form fields in declared order.
- MVP form field types: `text`, `textarea`, `choice` (`options: [...]`), `number`.
- **In-memory:** flat `Vec<Snippet>` + Aho-Corasick automaton over all triggers.
- **Validation:** per-file isolation; a broken file is logged and badged in settings, others still load.

## 3. Keystroke pipeline & expansion

```
WH_KEYBOARD_LL  →  hook thread  →  channel  →  matcher  →  injector
   (Win32)         (ring buffer)              (Aho-Corasick)  (SendInput)
```

**Hook thread:**
- `SetWindowsHookEx(WH_KEYBOARD_LL, …)` + `GetMessage` pump.
- Callback translates `VK` + modifiers via `ToUnicodeEx`, pushes onto a lock-free SPSC ring, returns. No alloc, no log, no block.
- Caps Lock, dead keys, IME composition: detected and ring reset (no expand mid-IME).

**Matcher:**
- 64-char rolling buffer per focus session. Cleared on focus change (`WinEventHook EVENT_SYSTEM_FOREGROUND`), mouse click, arrow/Home/End/PageUp/Down.
- Backspace pops one char.
- After each printable char, Aho-Corasick runs over buffer tail. Match fires when its trigger ends at buffer's end AND is preceded by start / whitespace / punctuation (configurable word-boundary).
- Greedy resolution: longest matching trigger wins.

**Injector:**
1. Suspend matcher (flag — buffer still updates, but no re-fire mid-injection).
2. `SendInput` Backspace × `trigger.len()`.
3. Resolve placeholders (form/shell may block — user sees form/spinner).
4. Inject via **clipboard paste** (save clipboard → write Unicode → `SendInput` Ctrl+V → restore). Fallback to per-char `SendInput` Unicode events if clipboard is locked or content > N KB.
5. Caret positioning: if `$|$` was present, send `Left` × (chars after token).
6. Resume matcher.

**Why clipboard paste:** orders of magnitude faster for long snippets; preserves indentation; sidesteps IME; one undo unit in most apps. Tradeoff: needs clipboard save/restore + ~10 ms settle. Per-char fallback for hostile apps.

**Safety rails:**
- Global pause hotkey (default `Ctrl+Alt+Pause`) toggles matcher without unloading the hook.
- Hardcoded denylist for password managers / UAC / secure desktop — buffer never accumulates while focused. MVP list: 1Password, KeePass, Bitwarden, LastPass, Windows UAC.
- Max expansion length cap (default 32 KB).

## 4. Forms, shell snippets, tray & autostart

### Dynamic forms

When a triggered snippet has any `type: form` var:
- Matcher pauses. Backspaces are **not** sent yet (cancel = clean state).
- Small Tauri webview window opens centered on focused monitor: `decorations: false, always_on_top: true, skip_taskbar: true`, ~400 px wide, auto-grows.
- Fields built from snippet's `vars`. Submit (Enter) → all placeholders resolve, previously focused HWND is reactivated (`SetForegroundWindow`), standard injection runs. Cancel (Esc) → nothing sent, buffer cleared, literal trigger stays.
- Form route is part of the settings UI bundle (`/form/<snippet-id>`).

### Shell-command snippets

- `cmd` is **argv array**, parsed into `Command::new(args[0]).args(&args[1..])`. String form refused — no `cmd.exe /c`, no injection surface.
- `timeout_ms` mandatory; default 1500, hard cap 10 000. Timeout → fail + tray notification.
- stdout trimmed, UTF-8 only.
- cwd = snippet folder; env inherited with only `PATH` (no incidental secret leak).
- **First-run consent dialog** the first time any shell snippet loads; pref persisted in `config.toml`. Off by default — snippets load but never execute until enabled.
- Per-snippet `confirm: true` for yes/no toast before each run.

### Tray & autostart

- Tray icon always present. Left-click → settings. Right-click menu:
  - Pause / Resume expansions
  - Reload snippets
  - Open snippets folder
  - Open settings
  - Quit
- Autostart via `HKCU\…\Run` registry value (handled by `tauri-plugin-autostart`). No service, no scheduled task — works without admin.
- Single-instance lock: named mutex `Global\openmacro-singleton`.

## 5. Cloud sync, testing, milestones

### Cloud sync options

| Option | Build | Tradeoff |
|---|---|---|
| **A. Git-backed (chosen for MVP)** | `sync/` subfolder is a git repo. App runs `git pull --rebase` / commit / push on a timer + on change. User points at any remote. | Zero backend. Free for users. Git 3-way merge handles YAML conflicts. Cost: user creates repo + PAT once. |
| B. Cloud-drive piggyback | Detect OneDrive/Dropbox folder, place snippets there. | Zero code. But vendor controls conflict resolution. |
| C. First-party backend | Auth'd REST or CRDT sync service. | Best UX. Largest scope: auth, hosting, billing, support, GDPR. |

A behind a `SyncBackend` trait so B/C can slot in later.

**Sync agent (option A):**
- Owns only the `sync/` subfolder; rest of `snippets/` is local-only.
- Tokio task, every 60 s and on debounced file change.
- `git fetch` → fast-forward if clean / commit local dirt → `pull --rebase=merges` → on conflict, abort rebase, copy conflicted files to `sync/.conflicts/<ts>/`, tray notification with one-click "open conflicts" → `git push`.
- Auth: HTTPS PAT in **Windows Credential Manager** (`wincred` crate). SSH supported via user's `ssh-agent`.

### Testing strategy

| Layer | Tooling | Coverage |
|---|---|---|
| Pure logic | `cargo test` | Aho-Corasick matcher, placeholder resolver, YAML parser, form value mapping, cursor token math, word-boundary rules. ≥80 % line coverage on these modules. |
| Hook + injection | Scripted harness | Spawn Notepad child, drive `SendInput`, assert via UI Automation. Windows CI runner. |
| Forms / UI | Vitest + Testing Library (template default) | Settings UI + form window component tests. Visual snapshots optional. |
| Sync | Integration | Local bare git repo (`tempdir`), two app instances, assert convergence + conflict shape. |

### Phased milestones (handoff to `writing-plans`)

1. **Skeleton** — Tauri scaffold, tray, autostart, settings webview shell, single-instance lock. *(Gemini front + Claude wiring.)*
2. **Storage + matcher** — YAML loader, file watcher, Aho-Corasick matcher, in-memory model, unit tests. *(Codex.)*
3. **Hook + injection** — WH_KEYBOARD_LL thread, clipboard-paste injector, pause hotkey, password-manager denylist, Notepad integration test. *(Codex.)*
4. **Placeholders + cursor token** — date / clipboard / cursor resolvers, end-to-end expansion. *(Codex.)*
5. **Settings UI** — snippet list, editor, broken-file badges, pause toggle. *(Gemini.)*
6. **Forms** — form window, field renderer, submit/cancel, focus restoration. *(split: Gemini front + Codex back.)*
7. **Shell snippets** — argv runner, timeout, first-run consent, per-snippet confirm. *(Codex.)*
8. **Sync** — git-backed agent, credential storage, conflict folder, tray notifications. *(Codex.)*
9. **Polish + installer** — MSI/MSIX packaging, signing, crash reporting, release pipeline. *(Codex.)*

### Risks / open questions

- **Code signing** — unsigned `.exe` installing a keyboard hook will hit SmartScreen + AV. Budget an EV cert (~$300/yr) before public release, or document the SmartScreen-bypass UX.
- **AV false positives** — global keyboard hooks look like keyloggers. Pre-release submission to Microsoft Defender + major AV vendors.
- **Exclusive-fullscreen / anti-cheat games** — `SendInput` may be filtered. Acceptable MVP gap; document.
